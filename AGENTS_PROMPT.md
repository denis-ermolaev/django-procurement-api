# 0. Инструкции ----
Если тебе сказали выполнить задачи из данного документа, то тебе нужно использовать файлы всего проекта, ты не должен что-то пропустить. Тебе разрешено делать глобальные изменения.

Создай команду разработчиков:
1. Первый должен придумывать планы, корректировать его, изменять и дорабаотывать техническое задание(`AGENTS_SPEC.md`), а также выдавать задание разработчикам.
2. Второй должен вести разработку
3. Третий выполнять контроль качества, быть критиком и тестировщиком.


Твой рабочий день 3 часа. Производи разработку проекта, всё это время итерактивно. После подведи результаты рабочего дня.

В начале придумай план, как ты будешь это всё выполнять, а после действуй в соответствии с ним

Возможно тебе потребуется сделать серьёзную переработку прокета, поэтому ты можешь сохранять, как план, так и промежуточные результаты, решения и нужную информацию в папку `.agents/` в Markdawn файлах, откуда потом ты можешь всё это читать.

Проверь файл `AGENTS_SPEC.md`, чтобы в нём было всё логично, для реализации api для закупок, если что измени, поправь, распиши подробнее.

# 1. AGENTS_SPEC.md ----
Проверь и если ещё не реализовано, то реализуй всю логику из файла `AGENTS_SPEC.md`, постепенно убирая весь legacy-код

# 2. Комментарии и logging ----
Они должны быть в правильном формате в соответствии с инструкциями `AGENTS.md`
Проверь, что во всех файлах комментарии в правильном формате, что сложные моменты в коде пояснены

logging должен зайдействовать все уровни, по умолчанию в настройках он должен быть отключён, сделай так, чтобы разработчику самостоятельно приходилось через .env включать логгирование необходимых модулей или функций. Если для чего-то это не реализовано, реализуй и придумай систему умного логирования

# 3. Makefile ----
- По максимуму используй докер контейнер для всех команд, исправь команды в соответствии с этим
- Внеси самые часто используемые команды

# 4. env ----
Должен быть приведён в формат `.env.example`, однако сами значения настроек не изменяй в `.env`

# 5. Тесты ----
- Создай в `docs/` файл `TEST.md`, где пропиши все необходимые тесты для полной проверки, что бизнес логика из `AGENTS_SPEC.md` абсолютно корректна, api работает
- Соотвественно приведи сами тесты в соответствии с этим файлом

# 5. docs/ ----
- Напиши файл docs.md с архитектурой документации, что нужно задокументировать, какой должна быть структура и т.п. Обязательные разделы: `env.md` с описанием всех переменных, как их получить и т.п
- После обнови и заполни документацию, чтобы она отражала действительный код
